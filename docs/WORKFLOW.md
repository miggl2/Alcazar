# AI 작업 흐름

사용자가 직접 관리하는 문서가 아니다. AI가 작업 중 까먹지 않기 위해 쓰는 절차다.

## 표준 순서

코드 작성 전후 순서는 아래 하나만 따른다.

1. 요청을 항목화한다.
2. 각 항목을 세 그룹으로 나눈다.
   - 답변됨: 이미 명확함. 추가 질문하지 않는다.
   - 추정됨: AI가 합리적으로 추정한 값. 맞는지만 묻는다.
   - 미답변: 결과가 크게 달라지는 항목. 답변을 받을 때까지 묻는다.
3. 사용자에게 의도 확인을 받는다.
4. `.intent/user-confirmation.md`에 확인 기록을 남긴다.
5. 필요한 메뉴얼만 읽고 `.intent/context.md`에 기록한다.
6. `.intent/plan.md`에 계획, 예상 수정 파일, 위험 요소를 쓴다.
7. 위험하거나 5개 초과 파일이 필요한 작업은 계획에 대한 사용자 확인을 추가로 받는다.
8. 코드 작성 후 `.intent/changed-files.md`에 실제 수정 파일과 이유를 기록한다.
9. `npm run check`를 실행하고 결과를 보고한다.

## poor prompt 대응

모호한 요청을 받아도 즉시 코드 작성 금지. 사용자가 양식을 직접 채우게 하지 말고 AI가 정리한다.

확인 문장은 아래 형식을 사용한다.

```text
이렇게 이해했습니다.
- 원하는 것:
- 원하지 않을 것 같은 것:
- 영향받는 기능/파일:
- 제가 추정한 것:
- 아직 답이 필요한 것:

맞으면 “응”, 다르면 다른 점을 말해주세요.
```

## 계획-검토-문서화

AI가 아래 파일을 관리한다.

- `.intent/current.md` — 원 요청과 의도 해석
- `.intent/plan.md` — 수정 목표, 예상 파일, 위험 요소
- `.intent/context.md` — 읽은 메뉴얼, 결정 이유, 작업 중 메모
- `.intent/checklist.md` — 이번 작업 전용 체크리스트
- `.intent/changed-files.md` — 실제 수정 파일 누적 기록
- `.intent/user-confirmation.md` — 사용자 확인 기록
- `.intent/package-checks.md` — 새 패키지 또는 버전 변경 검증 기록

## 완료 전 검증

1. `npm run check` 실행
2. 실패는 수정. 큰 실패는 전문 에이전트 보고서 작성
3. 변경 파일 한국어 3줄 요약
4. 사용자가 브라우저에서 직접 확인할 방법 안내
